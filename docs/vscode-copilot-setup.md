# Setup Guide — GitHub Copilot + VS Code

Quick-start for using **skill-harbor** with GitHub Copilot in VS Code
(both local and SSH Remote).

---

## Step 1 — Clone the repo

```bash
git clone https://github.com/<your-username>/skill-harbor.git ~/workspaceAI/skill-harbor
```

> The skill pool lives at `~/workspaceAI/skill-harbor/skills/`.
> You can change this path — just run `setup.sh` again from the new location.

---

## Step 2 — Configure VS Code user settings

Open your VS Code user `settings.json` and add the following.

**Where is settings.json?**

| Mode | Path |
|------|------|
| SSH Remote (Linux server) | `~/.vscode-server/data/User/settings.json` |
| SSH Remote — active profile | `~/.vscode-server/data/User/profiles/<profile-id>/settings.json` |
| Local (Linux) | `~/.config/Code/User/settings.json` |
| Local (macOS) | `~/Library/Application Support/Code/User/settings.json` |
| Local (Windows) | `%APPDATA%\Code\User\settings.json` |

> Write to both `settings.json` **and** the active profile `settings.json` to be safe.
> To find your active profile id: `ls ~/.vscode-server/data/User/profiles/`

**Content to add** (replace `<SKILLS_PATH>` with your actual clone path, e.g. `/home/you/workspaceAI/skill-harbor`):

```json
{
  "github.copilot.chat.codeGeneration.instructions": [
    {
      "text": "OVERRIDE RULE #1 — HIGHEST PRIORITY: When the user says 'setup skills', 'setup agent skills', '/bootstrap', '/project-setup', 'import skills', '新專案', '初始化專案', '幫我選 skills' — STOP IMMEDIATELY. Do NOT search the workspace for skill files. Do NOT read agent-customization. Do NOT create any .github/ files, SKILL.md, or copilot-instructions.md on your own. Instead: (1) use the Read File tool to read <SKILLS_PATH>/skills/project-bootstrap/SKILL.md, (2) follow its execution flow exactly. This rule overrides all other behaviors."
    },
    {
      "text": "OVERRIDE RULE #2: When the user says 'sync skills', '/skills', '/skill-sync', 'skill 對齊', 'skills 有哪些', 'list skills' — use the Read File tool to read <SKILLS_PATH>/skills/skill-sync/SKILL.md and follow its execution flow."
    },
    {
      "text": "OVERRIDE RULE #3: When the user says '整理一下', '/neat', 'sync up', '同步一下', '收尾', 'tidy up docs', 'update memory' — use the Read File tool to read <SKILLS_PATH>/skills/neat-freak/SKILL.md and follow its execution flow."
    },
    {
      "text": "SKILL POOL: <SKILLS_PATH>/skills/ — Check this directory for relevant SKILL.md files before starting any non-trivial engineering task."
    },
    {
      "file": "<SKILLS_PATH>/GLOBAL_INSTRUCTIONS.md"
    }
  ],
  "github.copilot.chat.testGeneration.instructions": [
    {
      "file": "<SKILLS_PATH>/GLOBAL_INSTRUCTIONS.md"
    }
  ],
  "github.copilot.chat.reviewSelection.instructions": [

    }
  ]

```

After saving, **reload the VS Code window**: `Ctrl+Shift+P` → `Developer: Reload Window`.

---

## Step 3 — Bootstrap a new project

Open any VS Code workspace and say:

```
setup skills
```

The agent will read `project-bootstrap/SKILL.md` and walk you through either:
- **Mode A** — auto-analyzing your project's docs/spec and recommending skills
- **Mode B** — listing all available skills for manual selection

It will then copy the selected skills into `.github/skills/` and create
`.github/copilot-instructions.md` so future sessions in this project also load them.

---

## Step 4 — Keep skills in sync

When you add new skills to the pool, run:

```
sync skills
```

This reads `skill-sync/SKILL.md` and updates the project's `using-agent-skills/SKILL.md`
registry to reflect what's actually installed.

---

## Key trigger words (reference)

| Say this | What happens |
|----------|-------------|
| `setup skills` | Runs project-bootstrap (Mode A or B) |
| `sync skills` / `/skills` | Runs skill-sync — aligns registry vs disk |
| `整理一下` / `/neat` / `sync up` | Runs neat-freak — end-of-session doc cleanup |

---

## Files that don't go in the repo

These are machine-specific and contain absolute paths — **do not push**:

- `~/.vscode-server/data/User/settings.json`
- `~/.vscode-server/data/User/profiles/*/settings.json`
- `<workspace>/.github/copilot-instructions.md` (per-project, generated by bootstrap)
